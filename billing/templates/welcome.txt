Quick start — prove a secret is in range [0, 100]:

  curl -X POST https://api.tinyzkp.com/prove/template/range_proof \
    -H "Authorization: Bearer {api_key}" \
    -H "Content-Type: application/json" \
    -d '{{"params":{{"min":0,"max":100,"witness_steps":[42,44]}}}}'

Verify the proof (always free):

  curl -X POST https://api.tinyzkp.com/verify \
    -H "Authorization: Bearer {api_key}" \
    -d '{{"proof": YOUR_PROOF_OBJECT}}'

What you can do:
  - 6 proof templates: range_proof, hash_preimage, accumulator_step,
    computation_attestation, policy_compliance, data_integrity
  - Zero-knowledge mode: add "zk_mask_degree": 1 to hide computation
  - Cost estimation: POST /estimate (no auth required)
  - Batch proving: POST /prove/batch (up to 100 jobs at once)
  - EVM calldata: GET /proof/:job_id/calldata for on-chain verification
  - Client-side verification: npm install @tinyzkp/verify (WASM, no server needed)

Key rotation:
  If your key is compromised, rotate it immediately:
  curl -X POST https://tinyzkp.com/api/rotate-key \
    -H "Authorization: Bearer {api_key}"

Keep your API key secret. Do not expose it in client-side code.
