You're in!

Check your email for your API key. Here's how to make your first proof.

1 Check your email

We sent your API key to your email. It looks like tzk_.... Store it securely — it’s only shown once.

Don't see it? Check spam, or contact support. Need to rotate a compromised key? See key rotation.

2 Submit a proof

Let’s prove that you know a secret number between 0 and 100 — without revealing the number itself. The witness_steps are internal computation values that encode your secret; the verifier never sees them.

curl -X POST https://api.tinyzkp.com/prove/template/range_proof \ -H "Authorization: Bearer tzk_YOUR_KEY" \ -H "Content-Type: application/json" \ -d '{"params":{"min":0,"max":100,"witness_steps":[42,44]}}'

Response: {"job_id":"prf_a1b2c3","status":"proving","eta_ms":1200}
Save the job_id — you’ll need it in the next step.

3 Poll until complete

Proof generation takes 1–5 seconds. Poll the job until status is "completed":

curl https://api.tinyzkp.com/prove/prf_a1b2c3 \ -H "Authorization: Bearer tzk_YOUR_KEY"

Response when done: {"status":"completed","proof":{"version":4,"bytes":"0x6a8f..."}}
If status is still "proving", wait a second and try again. Copy the entire proof object for the next step.

4 Verify it (free)

Pass the proof object from step 3 to the verify endpoint. Verification is always free — auth is required only to prevent abuse:

curl -X POST https://api.tinyzkp.com/verify \ -H "Authorization: Bearer tzk_YOUR_KEY" \ -H "Content-Type: application/json" \ -d '{"proof":{"version":4,"bytes":"0x6a8f..."}}'

Response: {"valid":true,"verified_in_ms":2.8} — The verifier confirmed the proof is valid without learning your secret number.
